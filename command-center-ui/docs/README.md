## Documentation Standards

### Creating New Documentation

1. **Choose the right category**:
   - `features/` - Feature specifications and guides
   - `integration/` - External service integrations
   - `implementation/` - Implementation plans and roadmaps
   - `testing/` - Testing strategies and guides
   - `security/` - Security documentation
   - `audits/` - System audits and reports

2. **Use markdown format**: All docs use GitHub-flavored markdown

3. **Link generously**: Cross-link related documentation

4. **Keep it updated**: Update the index when adding new docs
